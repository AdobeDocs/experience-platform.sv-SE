## Uppdaterat användargränssnitt {#ux}

Adobe Experience Platform släpper uppdateringar i domän- och huvudfältet för att förbättra din upplevelse och göra den enhetlig med andra [!DNL Experience Cloud]-program.

- Enklare att växla mellan olika organisationer eller till ett annat program
- Förbättrad användarhjälp, inklusive aktuella artiklar och sammanhangsberoende dokumentation på Hjälp-menyn
- Möjlighet att ge feedback om [!DNL Experience Platform] och supportärenden

Utbyggnaden av den nya upplevelsen är gradvis. Du kan visa upplevelsen på [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] använder maskininlärning och artificiell intelligens för att få insikter från era data. [!DNL Data Science Workspace] är integrerat i Adobe Experience Platform och hjälper er att göra prognoser med ert innehåll och era dataresurser över alla Adobe-lösningar. Ett sätt som [!DNL Data Science Workspace] gör detta på är genom att använda [!DNL JupyterLab]. [!DNL JupyterLab] är ett webbaserat användargränssnitt för  [[!DNL Project Jupyter]](https://jupyter.org/) och är nära integrerat med Adobe Experience Platform. Den utgör en interaktiv utvecklingsmiljö för datavetare som kan arbeta med [!DNL Jupyter] bärbara datorer, kod och data.

**Nya funktioner**

| Funktion | Beskrivning |
|--- | ---|
| [!DNL JupyterLab Launcher] | [!DNL JupyterLab Launcher] innehåller nu startsidor för [!DNL Spark] 2.4-anteckningsböcker. [!DNL Spark] 2.3 startsidor för bärbara datorer är nu markerade som inaktuella och inställda på att tas bort i en senare version. |
| [!DNL Spark] 2.4 | Nya recept från Scala ([!DNL Spark]) och PySpark använder nu [!DNL Spark] 2.4. |
| Kernlar | Scala ([!DNL Spark]) bärbara datorer har nu skapats via Scala-kernel. PySpark-anteckningsböcker skapas nu via kerneln [!DNL Python]. Kerneln [!DNL Spark] och PySpark är föråldrade och inställda på att tas bort i en senare version. |
| Recept | Nya PySpark- och [!DNL Spark]-recept följer nu [!DNL Docker]-arbetsflödet som liknar [!DNL Python]- och R-recept. |

Mer information om hur du migrerar anteckningsböcker och recept till [!DNL Spark] 2.4 finns i [guiden för dataåtkomst till anteckningsboken](../../data-science-workspace/jupyterlab/access-notebook-data.md). Mer allmän information om arbetsytan Datavetenskap finns i [översiktsdokumentationen](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

I [Kunddataplattform för realtid](../../rtcdp/overview.md) är mål färdiga integreringar med målplattformar som aktiverar data till dessa partners på ett smidigt sätt.

**[!DNL Facebook]**

CDP har nu stöd för dataaktivering i realtid till [!DNL Facebook], vilket gör att du kan aktivera profiler för dina [!DNL Facebook]-kampanjer för målgruppsanpassning, personalisering och undertryckning baserat på hash-kodade e-postmeddelanden.

Mer information om de nya funktionerna finns på sidan [Facebook-mål](../../destinations/catalog/social/facebook.md).

## [!DNL Experience Platform Web SDK] och [!DNL Experience Platform Edge Network]{#edge}

Med [!DNL Experience Platform Web SDK] och [!DNL Experience Platform Edge Network] kan användare skicka data till Adobe Experience Platform och andra Adobe Solutions i realtid för slutanvändarenheter och webbläsare. Den senaste listan över användningsfall finns i vår [allmänna färdplan](https://github.com/adobe/alloy/projects/5) som uppdateras ofta.

**Nya funktioner**

| Funktion | Beskrivning |
|--- | ---|
| Stöd för ECID | SDK har stöd för ECID direkt utan ytterligare bibliotek eller information att installera. |
| Konfigurationsgränssnitt | Hantera dina konfigurations-ID-inställningar med det nya edge-konfigurationsgränssnittet i [!DNL Launch], du måste vara på tillåtelselista för att få åtkomst. |
| Adobe Experience Platform Web SDK Mixin | En blandning som används med [!DNL Experience Platform web SDK] som omfattar alla fält som stöds. |
| Kontroll av kursens samtycke | Ger företag kontroll över deltagande och avanmälan av [!DNL Experience Platform Web SDK]. |
| Stöd för felsökning på klientsidan i nya [!DNL Experience Cloud Debugger Extension] | Se förfrågningar från [!DNL Experience Platform web SDK] och kantspårningar för att se hur data flödar genom systemet. |
| Adobe Analytics | Skicka data till [!DNL Analytics] rapportsviter via edge-konfigurationen. XDM förenklas till kontextdata och stöder taggning för flera programsviter. |
| Adobe Target | Stöd för Adobe Target. Inklusive VEC, formulärbaserad disposition, A/B, XT, Automated Personalization, MVT. |
| Adobe Audience Manager Support | Stöd för [!DNL Audience Manager] ID-synk, URL-mål och cookie-mål. |

Mer information om [!DNL Platform Web SDK] och [!DNL Edge Network] finns i [dokumentationen](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Med Adobe Experience Platform kan ni skapa samordnade, enhetliga och relevanta upplevelser för era kunder oavsett var och när de interagerar med ert varumärke. Med [!DNL Real-time Customer Profile] kan ni se en helhetsbild av varje enskild kund som kombinerar data från flera kanaler, inklusive online-, offline-, CRM- och tredjepartsdata. [!DNL Profile] kan ni sammanställa era olika kunddata i en enhetlig vy som ger ett användbart, tidsstämplat konto för varje kundinteraktion.

**Nya funktioner**

| Funktion | Beskrivning |
| -----------| ---------- |
| Nya mått för profilexport | Tillagda mätvärden för profilexportjobb som visar totalt antal exporterade profiler och antal profiler i varje namnutrymme. |
| Nya Insikter-mått för observationer | API:t för observationsinsikter har nu följande mått för direktuppspelning i [!DNL Profile]: Inkommande begärandefrekvens, lyckad överföringshastighet, inspelningsstorlek. |
| Slutpunkt för massutskick av GET | Massresultatslutpunkt har lagts till i [!DNL Real-time Customer Profile] API för att möjliggöra hämtning av flera GETTER i ett enda API-anrop. Nu kan du gruppvisa GET med upp till 100 ID:n för segmentdefinitioner, segmentjobb och sammanfogningsprinciper. |
| Bläddra profil efter identitet | I [!DNL Platform]-gränssnittet kan du nu välja ett identitetsnamnutrymme och ange ett identitetsvärde för att kunna bläddra i en profil. |

## Källor {#sources}

Adobe Experience Platform kan importera data från externa källor och samtidigt strukturera, etikettera och förbättra dessa data med hjälp av [!DNL Platform]-tjänster. Du kan importera data från en mängd olika källor som Adobe-program, molnbaserad lagring, tredjepartsprogram och ditt CRM-system.

[!DNL Experience Platform] innehåller ett RESTful-API och ett interaktivt användargränssnitt som gör att du enkelt kan konfigurera källanslutningar för olika dataleverantörer. Dessa källanslutningar gör att du kan autentisera och ansluta till externa lagringssystem och CRM-tjänster, ange tider för matning och hantera dataöverföringshastigheter.

**Nya funktioner**

| Funktion | Beskrivning |
| ------- | ----------- |
| Ytterligare API- och gränssnittsstöd för molnlagringssystem | Nya källanslutningar för [!DNL Azure File Storage]. |
| Ytterligare API- och gränssnittsstöd för databaser | Nya källanslutningar för [!DNL Azure Data Explorer], IBM DB2 och [!DNL Oracle DB]. |
| Adobe Audience Manager till [!DNL Experience Platform]-datadelning | Etableringsprocessen för [!DNL Audience Manager]-anslutningen har uppdaterats. [!DNL Audience Manager] datauppsättningar för  [!DNL Real-time Customer Profile] är nu inaktiverade som standard. Du kan välja vilka datauppsättningar som ska befordras till [!DNL Profile] manuellt. De nya standardinställningarna är inte retroaktiva och påverkar bara etablering för nya [!DNL Audience Manager]-anslutningar. Mer information finns i [användarhandboken för datauppsättningar](../../catalog/datasets/user-guide.md). |
