## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] använder maskininlärning och artificiell intelligens för att få insikter från era data. Integrerat i Adobe Experience Platform hjälper er att [!DNL Data Science Workspace] göra prognoser med hjälp av ert innehåll och era dataresurser i alla Adobes lösningar. Ett sätt [!DNL Data Science Workspace] att uppnå detta är genom att använda [!DNL JupyterLab]. [!DNL JupyterLab] är ett webbaserat användargränssnitt för <a href="https://jupyter.org/" target="_blank">[!DNL Project Jupyter]</a> och är nära integrerat med Adobe Experience Platform. Den utgör en interaktiv utvecklingsmiljö där datavetare kan arbeta med [!DNL Jupyter] bärbara datorer, kod och data.

**Nya funktioner**

| Funktion | Beskrivning |
|--- | ---|
| [!DNL JupyterLab Launcher] | I [!DNL JupyterLab Launcher] den finns nu startsidor för [!DNL Spark] 2.4 bärbara datorer. [!DNL Spark] 2.3 startsidor för bärbara datorer är nu markerade som inaktuella och inställda på att tas bort i en senare version. |
| [!DNL Spark] 2.4 | Nya recept från Scala ([!DNL Spark]) och PySpark använder nu [!DNL Spark] 2.4. |
| Kernlar | Scala ([!DNL Spark]) bärbara datorer har nu skapats via Scala-kernel. PySpark-anteckningsböcker skapas nu via [!DNL Python] Kernel. Kerneln [!DNL Spark] och PySpark är föråldrade och inställda på att tas bort i en senare version. |
| Recept | Nya PySpark och recept följer nu det arbetsflöde som [!DNL Spark] liknar recept [!DNL Docker] [!DNL Python] och recept. |

Mer information om hur du migrerar dina anteckningsböcker och recept till att använda [!DNL Spark] 2.4 finns i guiden [för migrering av](../../data-science-workspace/recipe-notebook-migration.md)anteckningsböcker. Mer allmän information om arbetsytan Datavetenskap finns i [översiktsdokumentationen](../../data-science-workspace/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Med Adobe Experience Platform kan ni skapa samordnade, enhetliga och relevanta upplevelser för era kunder oavsett var och när de interagerar med ert varumärke. Med [!DNL Real-time Customer Profile]det kan ni få en helhetsbild av varje enskild kund som kombinerar data från flera kanaler, inklusive online-, offline-, CRM- och tredjepartsdata. [!DNL Profile] kan ni sammanställa era olika kunddata i en enhetlig vy som ger ett användbart, tidsstämplat konto för varje kundinteraktion.

**Nya funktioner**

| Funktion | Beskrivning |
| -----------| ---------- |
| Nya mått för profilexport | Tillagda mätvärden för profilexportjobb som visar totalt antal exporterade profiler och antal profiler i varje namnutrymme. |
| Nya Insikter-mått för observationer | API:t för observationsinsikter har nu följande mått för direktuppspelning i [!DNL Profile]: Inkommande begärandefrekvens, lyckad överföringshastighet, inspelningsstorlek. |
| MassGET-slutpunkt | GET-slutpunkt för massutskick har lagts till i API för att det ska gå att hämta flera resultat i ett enda API-anrop. [!DNL Real-time Customer Profile] Du kan nu gruppvis få upp till 100 ID:n för segmentdefinitioner, segmentjobb och sammanfogningsprinciper. |
| Bläddra profil efter identitet | I [!DNL Platform] användargränssnittet kan du nu välja ett identitetsnamnutrymme och ange ett identitetsvärde för att kunna bläddra i en profil. |

## Sources {#sources}

Adobe Experience Platform kan importera data från externa källor och samtidigt strukturera, etikettera och förbättra dessa data med hjälp av [!DNL Platform] tjänster. Du kan importera data från en mängd olika källor, till exempel Adobe-program, molnbaserad lagring, programvara från tredje part och ditt CRM-system.

[!DNL Experience Platform] innehåller ett RESTful-API och ett interaktivt användargränssnitt som gör att du enkelt kan konfigurera källanslutningar för olika dataleverantörer. Dessa källanslutningar gör att du kan autentisera och ansluta till externa lagringssystem och CRM-tjänster, ange tider för matning och hantera dataöverföringshastigheter.

**Nya funktioner**

| Funktion | Beskrivning |
| ------- | ----------- |
| Ytterligare API- och gränssnittsstöd för molnlagringssystem | Nya källkopplingar för [!DNL Azure File Storage]. |
| Ytterligare API- och gränssnittsstöd för databaser | Nya källkopplingar för [!DNL Azure Data Explorer], IBM DB2 och [!DNL Oracle DB]. |
| Adobe Audience Manager till [!DNL Experience Platform] datadelning | Etableringsprocessen för [!DNL Audience Manager] kopplingen har uppdaterats. [!DNL Audience Manager] datauppsättningar för [!DNL Real-time Customer Profile] är nu inaktiverade som standard. Du kan välja vilka datauppsättningar du vill befordra till manuellt [!DNL Profile]. De nya standardinställningarna är inte retroaktiva och påverkar bara provisionering för nya [!DNL Audience Manager] anslutningar. Mer information finns i användarhandboken för [datauppsättningar](../../catalog/datasets/user-guide.md). |
